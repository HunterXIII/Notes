# Лекция 1 
**npm** - менеджер пакетов Node.js  
`npm install <package-name>` - установка всех пакетов из файла package.json  
- `--save` - позволяет установить пакет и добавить запись о нём в `dependencies` файла *package.json*
- `--save-dev` - в раздел для разработки (`devDependencies`)  
**Всегда создаём *package.json* с помощью `npm init`**  

- Исправление ошибки - patch
- Добавление функциональности, но при этом можно пользоваться старой версией - minor
- Добавление вообще новой функциональности, что на старой версии уже не получится работать - major  

## Модули
Каждый модуль отвечает за конкретную функциональность  
- Синхронная система модулей: CommonJS
- Асинхронная - ES Modules  
  
## Асинхронность
**Promise** - объект, представляющий результат асинхронной операции (успешный или неудачный)  
- async - объявляет асинхронную функцию
- await - приостанавливает выполнение функции по Promise не разрешится
- Для обработки ошибок используется try/catch, как в синхронном коде

## Работа с файловой системой
fs:
- Чтение, запись, удаление файлов и директорий
- Работа с метаданными (время создания, размер)  
path:
- Безопасная работа с путями в разных ОС
